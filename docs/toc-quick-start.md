## 2. 手动启动流程

### 2.1 步骤一：收集任务（2分钟）
1. 查看俊阳的最新需求
2. 查看 backlog（如有）
3. 查看子进程回报的问题
4. 为每个事项创建 TaskCard（使用模板）

**手动操作：**
```bash
# 创建示例 TaskCard
cp templates/task-card.yaml tasks/T-20260408-001.yaml
# 编辑 TaskCard 内容
```

### 2.2 步骤二：澄清任务（3分钟）
对每个 TaskCard，补充：

- goal：目标是什么
- definitionOfDone：完成定义
- dependencies：依赖
- risks：风险
- decisionNeeded：是否需要俊阳决策

**关键问题：**
- 这个任务的目标是否明确？
- 完成的标准是什么？
- 谁最适合做这个任务？
- 需要俊阳决策吗？

### 2.3 步骤三：识别主制约（2分钟）
检查当前状态：

1. **看队列**：哪个环节任务最多？
2. **看等待**：哪个环节等待时间最长？
3. **看返工**：有没有重复劳动？
4. **看中断**：哪个环节最常被打断？

**手动计算：**
- 数一数 `WaitingDecision` 状态的任务
- 数一数 `Verifying` 状态的任务
- 估算平均等待时间

### 2.4 步骤四：更新状态（1分钟）
更新 `state/constraint-record.yaml`：

- 更新 `evidence` 部分
- 更新 `symptoms` 部分
- 设置 `reviewAt`（下次检查时间）

## 4. 心跳集成

### 4.1 配置心跳
`HEARTBEAT.md` 已配置：

- 每 4 小时轻量检查
- 每日 2 次完整检查
- 每日日终复盘
- 每周周复盘

### 4.2 心跳时做什么
当收到心跳时：

1. 执行轻量检查（< 5分钟）
2. 如需要，执行完整检查
3. 记录检查结果
4. 返回 `HEARTBEAT_OK`

### 4.3 检查内容
**轻量检查：**
- 扫描 TaskCard 状态
- 检查 WIP 是否超限
- 快速判断主制约

**完整检查：**
- 计算所有指标
- 更新 ConstraintRecord
- 决定策略调整

## 5. 常见问题处理

### 5.1 没有 TaskCard 怎么办？
- 先创建 1-2 个示例 TaskCard
- 从当前正在做的任务开始
- 逐步建立习惯

### 5.2 数据计算太麻烦？
**初期简化：**
- 队列长度：数一数就行
- 等待时间：估算即可
- 返工率：记录返工次数
- 重点在趋势，不在绝对值

### 5.3 忘记执行怎么办？
- 设置简单提醒
- 从下一个心跳开始
- 不要追求完美，追求持续

### 5.4 俊阳不习惯怎么办？
- 先从小青侧开始
- 逐步引入 DecisionPacket
- 用实际效果证明价值

## 7. 成功标准

系统运行成功，如果：

1. **吞吐提升**：每周 Done 任务数增加
2. **库存下降**：等待决策/验证的任务减少
3. **返工减少**：重复劳动减少
4. **决策更集中**：俊阳的碎片决策减少

---

## 8. 紧急联系

如遇问题：

1. **系统不工作**：回退到简单清单模式
2. **数据混乱**：使用最近有效状态
3. **无法判断**：默认保持当前主制约
4. **完全卡住**：联系俊阳紧急处理

## 配置加载顺序

```
┌─────────────────────────────────────────────────────────┐
│                配置加载顺序（从低到高优先级）            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 内置默认值                                          │
│     ↓                                                  │
│  2. 共享基础配置 (shared/openclaw.base.json)           │
│     ↓                                                  │
│  3. 设备特定配置 (local/openclaw.local.json)           │
│     ↓                                                  │
│  4. 环境变量 (.env)                                    │
│     ↓                                                  │
│  5. 命令行参数                                         │
│                                                         │
│  优先级：命令行参数 > 环境变量 > 设备配置 > 共享配置    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 同步流程

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   设备 A        │    │    Git 仓库     │    │   设备 B        │
├─────────────────┤    ├─────────────────┤    ├─────────────────┤
│ 1. 检测更改     │    │                 │    │                 │
│    - 记忆更新   │    │                 │    │                 │
│    - 任务变更   │───▶│ 2. 提交到仓库   │───▶│ 3. 拉取更新     │
│    - 配置修改   │    │    - git add    │    │    - git pull   │
│                 │    │    - git commit │    │                 │
│ 4. 本地同步     │◀───│    - git push   │◀───│ 5. 应用更新     │
│    - rsync      │    │                 │    │    - rsync      │
│                 │    │                 │    │                 │
│ 6. 验证完整性   │    │ 7. 冲突解决     │    │ 8. 验证完整性   │
│    - 检查文件   │    │    - 手动合并   │    │    - 检查文件   │
│    - 测试功能   │    │    - 自动备份   │    │    - 测试功能   │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```


## 冲突解决策略

```
检测到冲突
    │
    ▼
┌──────────────┐
│ 冲突类型判断 │
└──────┬───────┘
       │
       ├───▶ 文本文件冲突 ───┐
       │                    │
       ├───▶ JSON 配置冲突 ─┼─▶ 自动合并（优先最新时间戳）
       │                    │
       └───▶ 二进制文件冲突 ─┘
                              │
                              ▼
                       ┌──────────────┐
                       │ 备份冲突文件 │
                       │  - 添加.conflict后缀 │
                       └──────┬───────┘
                              │
                              ▼
                       ┌──────────────┐
                       │ 应用合并结果 │
                       │  - 保留双方更改 │
                       └──────┬───────┘
                              │
                              ▼
                       ┌──────────────┐
                       │ 记录冲突日志 │
                       │  - 记录到sync.log │
                       └──────────────┘
```

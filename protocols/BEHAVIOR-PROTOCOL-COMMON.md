# 行为协议 - 通用约束

## 1. 单点定义约束
- **同一规则只定义一次**：同一行为规则、模板结构、字段说明、算法口径，只允许存在一个权威定义位置
- **非权威文件只做引用**：其他文件只保留最小必要的入口、索引或引用，不重复展开正文
- **实现细节不回流协议**：脚本、渲染、模板实现等细节应留在脚本或模板文件中，协议只保留规则与引用
- **发现重复时的默认处理**：优先保留最贴近职责边界的主定义，删除其余重复内容，而不是多处同步改写
- **修改顺序**：先定位权威源并修改，再检查引用是否仍然有效，不要在多个文件中并行重写同一信息

## 2. 协议变更授权约束

- **协议变更需授权**：任何对 `BEHAVIOR-PROTOCOL*.md` 文件的修改（包括编辑、添加、删除内容）都必须获得俊阳（用户）的明确授权。未经授权的协议变更不得执行。

## 3. 适用方式
- 所有进程协议默认继承本文件约束
- 如子协议未另行说明，默认遵循本文件
- 如子协议与本文件冲突，优先保留更贴近该角色职责边界、且重复度更低的定义，并同步清理重复项

## 4. 协议执行强制性
- **协议读取与遵守属于硬约束**：不是参考建议。
- **任何进程** 在做任何实质性动作前，都必须先完成适用于自身角色的协议读取。
- **未读取完成不得开始执行**：如果缺少预期协议文件，应暂停执行并上报，而不是跳过协议直接开工。
